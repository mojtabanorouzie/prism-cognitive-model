# چیت‌شیت (راهنمای سریع فارسی)

این‌ها دستور اسلش (slash command) نیستند؛ جمله‌های زبان طبیعی‌ای هستند که در گفتگوی اصلی Claude Code تایپ می‌کنی و به‌طور خودکار به عامل (agent) درست هدایت می‌شوند. معماری کامل پروژه در [README.md](README.md) توضیح داده شده.

از Claude Code استفاده نمی‌کنی؟ بخش «Option B — Any other model» در README را ببین؛ همین دستورها به‌صورت متن ساده در هر مدل دیگری (ChatGPT، Gemini، Claude.ai، یا حتی یک مدل لوکال) هم کار می‌کنند.

نسخه انگلیسی این فایل: [CHEATSHEET.md](CHEATSHEET.md)

> ⚠️ نام عامل‌ها (مثل `cognitive-systems-analyst`) باید دقیقاً به همین شکل انگلیسی نوشته شود تا Claude Code بتواند آن را به فایل درستش در `.claude/agents/` وصل کند. بقیه‌ی جمله را می‌توانی کاملاً فارسی بنویسی.

## چرخه اصلی

| هدف | چی تایپ کنم |
|---|---|
| **بررسی** یک رفتار | `از عامل cognitive-systems-analyst بخواه <رفتار> را بررسی کند` |
| **ثبت نهاییِ** یافته‌ها در مدل | `از عامل cognitive-historian بخواه آخرین session را reconcile کند` |
| **طراحیِ یک آزمایش** | `از عامل behavioral-experiment-designer بخواه برای H005 یک آزمایش طراحی کند` |
| **ثبتِ یک نتیجه** | `از عامل behavioral-experiment-designer بخواه نتیجه E001 را ثبت کند: پیش‌بینی 6، نتیجه واقعی 0` |
| **اعمالِ نتیجه** | `از عامل cognitive-historian بخواه تغییرات E001 را commit کند` |

مسیر کامل: **analyst → historian → designer → (خودت در زندگی واقعی اجرایش می‌کنی) → designer → historian.** هیچ‌چیز خودکار نیست؛ هر مرحله را خودت هدایت می‌کنی.

## عامل 1 — Analyst (بررسی)

```
از cognitive-systems-analyst بخواه بررسی کند چرا همیشه قبل از تحویل، یک ماژول را از نو می‌نویسم
از analyst بخواه H004 را به چالش بکشد — دنبال نمونه‌های نقض بگردد
از analyst در این‌باره بپرس: گفتم «من فقط تنبلم» راجع‌به رفتن به باشگاه
```
یک فایل session در `sessions/` می‌نویسد، همراه با تغییرات پیشنهادیِ ضریب اطمینان. این تغییرات را خودش commit **نمی‌کند**.

## عامل 2 — Experiment Designer (طراحی آزمایش)

```
از behavioral-experiment-designer بخواه برای رابطه‌ی H006/H007 یک آزمایش طراحی کند
از experiment-designer بپرس همین الان باارزش‌ترین آزمایش ممکن چیست
از experiment-designer بخواه نتیجه‌ی E001 را ثبت کند: پیش‌بینی 7، نتیجه واقعی 1
```
فایل‌های داخل `experiments/` را می‌نویسد یا به‌روزرسانی می‌کند. فرضیه‌های نزدیک به **0.50** را هدف می‌گیرد؛ چون بیشترین اطلاعات جدید را می‌دهند.

## عامل 3 — Historian (ثبت نهایی + خواندن وضعیت)

```
از cognitive-historian بخواه session با نام S-2026-07-16-seed را reconcile کند
از cognitive-historian بخواه H004 را reject کند و H005 را به‌عنوان والدش ثبت کند
از cognitive-historian بخواه MODEL.md را دوباره بسازد
از cognitive-historian بخواه داشبورد و تایم‌لاین فعلی را نشانم دهد
```
تنها عاملی است که فایل‌های `hypotheses/` و `MODEL.md` را می‌نویسد. هر تغییرِ ضریب اطمینان باید یک دلیلِ ثبت‌شده + منبع داشته باشد.

## خواندن مستقیمِ وضعیت (بدون نیاز به عامل)

```
MODEL.md را نشانم بده          # داشبورد — از همین‌جا شروع کن
H005 را نشانم بده
همه‌ی فرضیه‌های ردشده را فهرست کن
آزمایش‌های در حال اجرا را نشانم بده
```

## حرکت‌های ترکیبیِ کاربردی

```
یک چرخه‌ی کامل روی <رفتار> اجرا کن: اول analyst، بعد historian، بعد designer
همین الان باارزش‌ترین سوال بازِ من چیست؟
کدام فرضیه کمتر از همه تست شده؟          # دنبال بخش‌های خالیِ «Evidence against» بگرد
ضریب‌های اطمینان را از روی شواهد دوباره حساب کن و بگو چه چیزی تغییر کرد
```

## قوانین ثابت (برای تشخیص رفتار اشتباهِ یک عامل)

- Analyst و Designer فقط **پیشنهاد** می‌دهند؛ فقط **Historian** ثبت نهایی می‌کند. اگر analyst مستقیماً یک فایلِ hypothesis را edit کرد، یعنی پروتکل را زیر پا گذاشته.
- هیچ ضریب اطمینانی هرگز به **1.00** نمی‌رسد و هر تغییر باید منبع مشخصی داشته باشد (`S-…` یا `E…`).
- فرضیه‌های ردشده **بازنشسته می‌شوند، نه حذف** — اگر لیستِ `rejected` خیلی زود خالی بماند، یعنی هنوز چیزی رد نشده.

## اولین باره اینجایی؟

اگر داخل `cognitive-model/hypotheses/` فقط فایلِ `H000-example-*` هست، یعنی هنوز مدل خودت را نساخته‌ای. با این شروع کن:
```
از cognitive-systems-analyst بخواه <چیزی که واقعاً مدام درباره خودت متوجهش می‌شوی> را بررسی کند
```
